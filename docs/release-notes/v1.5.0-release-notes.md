# NakliData v1.5.0 — release notes

Tag `v1.5.0` at `origin/main`. The first surface of the **Resolve track** —
NakliData's sovereign answer to the agentic-CDP *resolve → segment → own* loop,
done locally and owned as files. This release is **M1: clustering / fuzzy-merge**.
Versioning is git-tag-based; the in-app `BUILD_VERSION` stays `0.1.0`.

(The v1.4.0→v1.5.0 delta also includes the v1.4.1 patch batch — F5/F6 + the
local-model load/registration fixes — see `plan/v1.4.1-release-notes.md`.)

## Headline — Cluster values (Resolve M1)

Detect variant spellings of a column's values (`Sharma Trading Co` =
`Sharma Trading Co.` = `SHARMA TRADING CO`) and emit an **additive** CASE-rewrite
SQL cell (`<col>__merged`) you run. The CASE cell is the artifact — reproducible,
replays via DuckDB with no model and no network.

- **Two OpenRefine-standard methods.** **Key collision** (fingerprint:
  trim → lowercase → NFKD diacritic-fold → strip punctuation → token sort+dedupe)
  is the safe, threshold-free default. **Nearest neighbour** (normalized
  Levenshtein) is the opt-in "find typos" pass, with a similarity slider
  (0.70–0.95, default 0.85). NN is blocked (first-fingerprint-char + an exact
  length-window prune) and capped at 5,000 distinct values.
- **Two entry points.** A "Cluster" chip on every SQL result, and a per-column
  "Cluster values" action in the schema panel.
- **Review before you run.** A modal shows each cluster with an editable
  canonical (default most-frequent → longest → lexicographic), per-variant row
  counts, and accept/reject (rejected dim, not hidden). Live CASE preview;
  empty-state and too-many-values guidance.
- **Injection-safe by construction.** Reuses the v1.2 M5 emitter — every variant
  value + canonical flows through `quoteLiteral`, the column + alias through
  `quoteIdent`. Hostile idents/values can't escape quoting (covered by a
  dedicated test battery).

## The removable AI — sidecar Job 8 `propose-merge`

An optional **Ask AI to check ambiguous pairs** affordance (shown only when a
sidecar provider is configured) adjudicates the borderline pairs the
deterministic pass didn't group (similarity in `[t-0.1, t)`). Structured
decision only — **no prose** (three-layer guard like propose-chart) — with a
**per-pair allowlist** hallucination guard that blocks both fabricated values and
recombined pairings. Rides the existing BYOK + local provider ladder; no new
egress. **Removability holds:** delete the job and key-collision + NN still
cluster fully.

## Compatibility

- **No `.naklidata` schema change** — clusters are ephemeral UI state; the
  durable output is an ordinary SQL cell that already round-trips. Pre-M1 files
  are unaffected.
- **One new dependency:** `fastest-levenshtein` (~2 KB, MIT).
- **No breaking changes.**
