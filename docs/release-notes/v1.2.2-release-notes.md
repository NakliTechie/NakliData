## Headline security fixes

### XSS in the Suggested Reports panel (CRITICAL → closed)

Untrusted column / table names from mounted xlsx / CSV / parquet
files were interpolated directly into innerHTML by
`renderTemplateCard`. A header like `<img src=x onerror=...>` fired
JS as soon as classification surfaced the column. XSS in NakliData
reaches BYOK keys in sessionStorage → exfil via `connect-src https:`.
Closed by routing every piece through `escapeHtml` via the new
`formatUsedColumnsHtml` helper. 7 vitest cases lock the contract.

### SSRF via `?lens=` shared links (HIGH → closed)

A malicious shared-link could enumerate any remote-source kind (HTTP,
Iceberg table, Iceberg catalog, S3 endpoint, Compute Bridge,
Compute Bridge catalog) and silently re-mount on page load. Used to
probe internal networks from the victim's browser + replay any
persisted bearer tokens against attacker-controlled URLs. Closed by
a new confirmation modal (`src/ui/lens-confirm-modal.ts`) that lists
every host the link would fetch from and requires explicit click.
Cancel is the default-focused button so Enter dismisses safely.
Local sources (example bundle, FSA folder) still auto-restore
silently.

### NL → SQL parser hardening (HIGH × 3 → closed)

Three parser bypasses found across audit + self-review:

1. **Comma-join allowlist bypass.** `FROM allowed, secret_table`
   slipped `secret_table` through the hallucination guard because the
   regex captured only the first ident after FROM. Replaced with
   `extractFromTables` — a positional walker that handles SQL-89
   comma-join + alias forms.
2. **Replacement-scan via single-quoted FROM.** `SELECT * FROM
   'https://attacker/x.csv'` (DuckDB's read_csv_auto shortcut)
   parsed to zero found tables and passed every guard. Closed by
   pre-scanning for `/\b(?:FROM|JOIN)\s+'/i`.
3. **Session-mutating keywords missing.** `SELECT 1; SET
   enable_external_access=true; SELECT 1` evaded WRITE_KEYWORDS.
   Added INSTALL / LOAD / SET / RESET / USE plus a multi-statement
   gate.

Adversarial review also caught two secondary bugs: false-rejections
on `LATERAL` / `UNNEST` queries, and false-rejections on `;` inside
string literals. Both fixed.

### CSP defence-in-depth (HIGH → closed)

Added `base-uri 'self'`, `object-src 'none'`, `form-action 'self'`,
`frame-ancestors 'none'` to the CSP. `base-uri` in particular closes
an exfil vector that `script-src` doesn't cover: an injected `<base>`
tag would redirect every relative URL (chunk loads, service worker
registration, duckdb-fallback) to an attacker origin.
`frame-ancestors` is documented as `<meta>`-ignored per CSP L3 —
ships anyway for the future when a header-capable deploy lands.

### BYOK-key exfil channels (MEDIUM × 3 → closed)

- New `providers/redact.ts` scrubs `Bearer`/`sk-…`/`sk-ant-…`/
  `x-api-key:` patterns from response error bodies before they reach
  the UI.
- Custom-endpoint URL is now hard-validated as parseable + https
  before any fetch + surfaced inline as a host chip (so the user can
  spot drift between typing and sending).
- `defaultTransport` no longer silently falls through to OpenAI for
  unknown providers — corrupted settings now throw loudly.

### Supply-chain pinning (HIGH → closed)

Postinstall scripts now treat the checked-in `integrity.json` as the
authoritative pinned-hashes table. Downloaded bytes are sha384'd and
compared against the pin; mismatch throws a "supply-chain alert".
Bootstrap path records new hashes only when no pin exists. Adversarial
review also added on-disk re-verification before the
`alreadyVendored()` shortcut — closes the in-place-tamper-between-
installs gap.

### Bearer-token CRLF rejected (MEDIUM → closed)

New `src/core/bearer-token.ts` validates against RFC 7235 token68
charset. Wired into `engine.configureIceberg` + `BridgeClient`
constructor. Rejects CR/LF / whitespace / quote characters with a
clear `InvalidBearerTokenError`. 9 vitest cases.

### Iceberg catalog scheme allowlist (HIGH → closed)

`mountIcebergCatalog` previously passed catalog-returned
`metadataLocation` straight to DuckDB without re-checking the URI
scheme. Codex caught a regex error (`https?://` accepted plain HTTP)
that defeated the first attempt at this fix. Now strictly
`^https://|^s3://`.

## Correctness fixes (selected)

- **`Math.min(...vals)` stack overflow.** Histogram / scatter / line
  chart min/max replaced with single-pass loops. No more
  `RangeError` on ≥125k-row numeric columns.
- **Templates panel chart binding.** `chart('bar', 'errors_by_service',
  ...)` previously ignored the second arg and bound by
  nearest-previous-named-SQL-cell. ERROR_FREQUENCY's bar chart
  silently rendered the time-series. Now resolves by name first.
- **Pivot + Map cells were unnameable.** PivotCellState /
  MapCellState carried a `name` field but the renderers didn't
  expose an editor for it; dashboards couldn't reference them.
  Added the input + change wiring.
- **Compare-tables column-alias collision.** Two columns differing
  only in non-alphanumerics (`"foo bar"` vs `"foo-bar"`) collapsed
  to the same sanitized alias. Switched to index-based aliases
  (`a_0`, `b_0`, …).
- **Service worker cache invalidation.** SW `CACHE_VERSION` was a
  hard-coded `'v1'`. Build now injects a 12-char hash-derived value;
  any change to the inline script bumps the SW cache. Stale-cache
  returning-user wouldn't have booted otherwise.
- **Taxonomy worker init.** Now has `error` + `messageerror` event
  listeners + 15s timeout + Worker.terminate on failure. A misconfigured
  deploy prefix no longer hangs the schema panel forever.
- **`buildSchemaHint` cross-table column duplication.** Filter by
  `${sourceId}::${tableId}::` key prefix; each table's hint now
  contains only ITS columns.

## Engineering: process notes

This release leaned heavily on **two-track adversarial review**:

1. A 5-subagent parallel forward-pass identified 33 issues in the
   v1.2.1 baseline (see `plan/forward-pass-2026-06-02.md`).
2. After all 33 were fixed, an internal code-reviewer agent + an
   external `codex review --base v1.2.1` independently audited those
   fixes. They found 9 NEW bugs the fixes themselves introduced.

The two reviewers had different blind spots:
- Internal agent caught: NL→SQL parser bugs, supply-chain hash gap,
  taxonomy worker leak, multi-statement gate string-literal trip,
  pendingMounts dead-code, CACHE_VERSION regex single-shot.
- Codex caught: iceberg catalog `http?://` regex error, sinks.ts
  SQL-vs-filename disconnect.

Same pattern that found 2 bugs after v1.2.0 and 5 bugs after Wave 5/6.
**External + internal review will be a standing post-batch gate
going forward.**

## Spec amendments

**None.** All fixes are correctness or hardening of existing surfaces.
The HIGH-severity SSRF fix (lens auto-mount → confirmation) DOES
change observable behavior of A15 (share-link round-trip), but the
contract — "shared link restores workbook state" — still holds; we
just add a one-click confirmation when remote sources are present.
Documenting as a behavior note in spec, not a contract change.

(If a future amendment review wants formal entries: A19 lens auto-mount
confirmation, A20 postinstall hash-pin protocol, A21 bearer-token
RFC 7235 charset, A22 CSP defence-in-depth set, A23 NL→SQL parser
safety guarantees. Captured in `plan/forward-pass-2026-06-02.md`
progress log but not yet formalised.)

## Breaking changes

**None.** Pure bug-fix release. Existing notebooks load and run
unchanged. Shared lens-link auto-mount adds a confirmation step for
remote sources only — local-source links (example bundle, FSA folder)
restore silently as before.
