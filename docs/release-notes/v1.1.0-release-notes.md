# NakliData v1.1.0 — release notes

**Tag date:** 2026-05-24
**Tagger commit:** e9182a7 (or HEAD at tag time)
**Previous tag:** v1.0.0 at `5b10b93` (2026-05-17)
**Scope:** 27 commits since v1.0.0.

---

## TL;DR

v1.1 turns NakliData from a notebook-and-classifier into a full semantic
workbench. The schema panel — the spec's single most important surface —
grows column profiling, table comparison, override rules, demo mode, and
a Cytoscape graph view. A BYOK AI sidecar adds three narrow, structured
jobs (explain query error, disambiguate type, define new type) that
help without ever narrating or auto-running SQL. Multi-session
workspaces, share links (`?lens=`), and PWA install land at the platform
layer. Two new cell kinds (pivot, map) and four new chart marks (pie +
stacked-bar / area-stacked / heatmap) round out the notebook. Two new
file formats (GeoJSON, KML) take the supported list from 13 to 15. The
taxonomy bundle grew from 40 to 41 types with a real CIN regex bug fix
along the way.

Privacy posture is unchanged: data never leaves the tab. The only
network egress added in v1.1 is the BYOK sidecar to your chosen provider
URL, and only when you opt in.

## What's new

### AI sidecar (BYOK, optional) — spec §4.3

Off by default. Open the gear icon → AI sidecar to enter a key. Three
narrow jobs:

1. **Explain query error** (`d2250a8`). Inline affordance on errored SQL
   cells. Prose explanation + optional "Copy SQL" suggested-fix block.
   Never auto-executed.
2. **Disambiguate type** (`0c83cbe`). Shown on schema-panel columns where
   two or more taxonomy candidates remain plausible (confidence
   ∈ [0.5, 0.9) and origin = detector). One-token answer matched against
   the candidate list; applied via the standard override path.
3. **Define a new type** (`b08d679`). In the "+ Define new type from this
   column…" modal, an optional "Suggest with sidecar" button fills
   regex / checksum / sql_type guesses. User edits and saves.

Providers shipped: **Anthropic, OpenAI.** Custom OpenAI-compatible
endpoints (local llamafiles, vLLM) are deferred to v1.2 — they need a
CSP rework first (today's explicit-host whitelist won't survive runtime
URL configuration).

**BYOK posture** (per spec amendment
[A2](./spec-amendments.md#a2--byok-key-persistence-amends-spec-4-hard-not-2)):
sessionStorage default + opt-in plaintext IDB with honest labelling and
a "Forget" affordance. Passphrase-encrypted persistence is parked for
v1.2.

**Classifier integration of user types** (`a71ebf7`). User-defined types
now feed the classifier directly: regex + header_match detectors are
synthesised per user type, the worker rebuilds the effective bundle on
`set_user_types`, and a "Re-classify with user types" button in the
schema panel preserves user accepts/overrides across re-runs.

### Schema panel — Theme 4

- **Column profile** (`a5e8f85`). Per-column toggle. `Engine.profileColumn`
  runs a full-table aggregate + top-5 in one query; renders inline under
  the column row.
- **Compare tables** (`0b14ff7`). Modal opens when ≥2 tables mounted.
  Auto-suggests join keys from shared semantic types. `Engine.compareTables`
  does FULL OUTER JOIN + bucket aggregate + per-row column-level diff
  sample.
- **Override rules** (`0b14ff7`). After a single-column override, a
  toast offers to promote the pick to a rule. Rules apply on every
  future mount + reclassify, with origin `user_override`. Manage-rules
  modal lists current rules; Remove button removes them. Persisted in
  `.naklidata` as `override_rules`.
- **Demo mode** (`0b14ff7`). Settings toggle. `maskLabel()` rewrites
  user paths and column names in the sources panel, schema panel, and
  result table headers (`src_1`, `tbl_1`, `col_1`, …). SQL cell text
  and data row values are not masked — those must be cleared manually
  before screenshots.
- **Schema graph** (`c8b4eef`). Cytoscape modal showing taxonomy-type
  relationships (e.g., `gstin` ↔ `pan` ↔ `vendor_name`), fed by
  `taxonomy/v0.1/relationships.json`. Lazy chunk; only loaded on demand.

### Sessions and sharing — Theme 3 wave 2

- **Multi-session** (`c9fcb48`). Header dropdown for create / switch /
  rename / delete. Per-session snapshot at `sessions/<id>/snapshot` in
  IDB. First-boot migration adopts the legacy `workbook/current` as the
  seed session. Auto-save flushes-then-flips on switch so in-flight
  debounced saves land on the outgoing session.
- **Share link `?lens=<base64>`** (`bf45db1`). Browser-native
  `CompressionStream('gzip')` + base64url; no new dependency. Share
  button in the header copies a URL to the clipboard. On boot, `?lens=`
  overrides the IDB workbook snapshot; on bad lens, falls back to IDB
  (not empty state). URL is stripped via `replaceState` after applying.
- **PWA installable** (`f81b660`). `manifest.webmanifest` + lite service
  worker. Precaches shell + chunks + manifest + icon + taxonomy worker
  on install; SWR for same-origin GETs; navigation fallback to cached
  `index.html` offline. DuckDB-fallback bytes (~74 MB) NOT precached;
  opportunistically cached on a one-time `?offline=1` warm-up.

### Notebook — Theme 2 + Wave 1 polish

- **Pivot cell** (`065b9f0`). New cell kind. In-memory pivot over the
  upstream SQL cell's `lastResult.rows` (no extra DuckDB query). Row,
  column, value, agg (`sum` / `avg` / `min` / `max` / `count`). Row +
  column + grand totals only when semantically meaningful. Display cap
  200 rows × 50 cols.
- **Map cell** (`06c11ee`). New cell kind. Tile-less MapLibre canvas
  (no basemap — privacy-clean). Reads a GeoJSON geometry column
  (object- or string-shaped). Optional categorical color via `match`
  expression. Three layers: polygons (+ outline), lines, points.
  Auto-fit bounds.
- **Observable Plot lazy chunk** (`2d258d7`). New chart marks:
  `stacked-bar`, `area-stacked`, `heatmap`. Plot lives at
  `dist/chunks/observable-plot.js` (~280 KB lazy) and never touches the
  shell.
- **Pie chart + faceted small-multiples** (`983827f`). Pie is a custom
  SVG arc renderer (Plot doesn't ship a pie mark by design). Aggregates
  per category, drops non-positive, caps at 12 + "Other" bucket. The
  third "facet-by" picker is shown only for facetable types (pie + the
  three Plot-rendered: stacked-bar, area-stacked, heatmap). Plot uses
  native `fy`; pie partitions into a grid (top-9 by total, remainder
  collapsed into a hidden count).

### Mount layer + offline

- **GeoJSON + KML mounts** (`06c11ee`). Via DuckDB spatial extension and
  `ST_Read`. `FileFormat` grows to `'geojson' | 'kml'`; file-picker
  accept list extended. Supported formats: **13 → 15.** Shapefile
  deferred — FSA single-file picker can't deliver multi-file `.shp /
  .dbf / .shx` bundles cleanly.
- **Vendored DuckDB extensions** (`d81881c`). `public/duckdb-extensions/v1.1.1/wasm_eh/`
  now ships:
  - `json` (680 KB) — unblocks the JSONL access-log mount in offline
    smoke. Smoke now asserts 4 mounted tables (was tolerant of 3).
  - `sqlite_scanner` (1.6 MB) + `sqlite` alias copy — extension loads
    but `ATTACH` fails due to a duckdb-wasm VFS bridge limitation (see
    DECISIONS 2026-05-23). Vendored for the day upstream fixes it.
  - `engine.boot({ offline: true })` sets
    `custom_extension_repository = '${origin}/duckdb-extensions'` so
    DuckDB picks up the vendored bytes.
  - `tests/e2e/offline-extensions.spec.ts` asserts zero
    `extensions.duckdb.org` fetches under `?offline=1`.
  - `excel` + `read_stat` deferred — not published at
    extensions.duckdb.org for v1.1.1/wasm_eh.

### Taxonomy — editorial pass

- Bumped from 40 → 41 types: added **UUID** (`bc78d4a`).
- Fixed real **CIN regex bug** — was `[LUu]`, corrected to `[LU]`. The
  third character of an Indian Corporate Identification Number can only
  be `L` or `U`, never `u`.
- Removed `iso3` from `iso_country_code` header_match — too noisy in
  practice; it matched too many spurious columns.

### Persistence / format

- `.naklidata` v1.0 gains two optional additive fields:
  - `user_types: UserType[]` — per-workspace taxonomy extensions
    (`b08d679`).
  - `override_rules: OverrideRule[]` — `0b14ff7`.
  Older readers round-trip both cleanly (additive only; format version
  unchanged at `1.0`).

## Bug fixes (carryover from v1.0 review)

- **CM6 EditorView lifecycle across notebook reloads** (`bc78d4a`).
  `notebook.load()` now disposes outgoing-set SQL cells' editors. Stops
  a slow memory creep across `.naklidata` loads + session switches.
- **SRI manifest round-trip coverage** (`bc78d4a`). New
  `tests/sri-integrity.test.ts` — 3 vitest specs incl. a negative tamper
  case. The browser's built-in SRI does the real check; the new tests
  cover the `integrity.json` ↔ vendored bytes round-trip, which was the
  only un-tested link in the chain.
- **Save-load flake confirmation** (review carryover, no commit). 5/5
  stable runs at ~2.0s each. No flakes seen.

## Internal / build

- **Latent CSP-hash bug in `esbuild.config.mjs`** (`7a73bc4`).
  `String.prototype.replace` was interpreting `$&` in the minified
  script body as the matched substring, drifting the inlined CSP hash
  off the actual bytes. Fixed with function-form replacers at three
  call sites. The Wave 1 pie + facet additions tipped the minified
  bundle across the threshold where `$&` first appeared, which surfaced
  the bug.
- **e2e save-load stability.** Test now uses
  `waitForClassificationStable` at both pre-save snapshot and
  post-load wait, and clears IDB between save + reload so the explicit
  Load click doesn't race boot-time auto-restore (both invoke
  `applyLoadedFile`, which isn't safe to interleave). The W1.3c "5/5
  stable" carryover confirmation was lucky timing; this fix removes
  the race. Underlying `applyLoadedFile` re-entrancy bug is left for
  a follow-up.
- **e2e override-rules toast wait bumped 4s → 8s** for workers=2
  contention headroom.

## Compatibility

- **`.naklidata` format:** version unchanged at `1.0`. New optional
  fields (`user_types`, `override_rules`) round-trip cleanly through
  v1.0 readers.
- **Browser support:** unchanged (Chrome / Edge / Opera 122+; Firefox
  partial; Safari unsupported).
- **Bundle budget:** shell `dist/index.html` at **413 KB** (+93 KB vs
  v1.0); well under the 600 KB spec gate.

## What's deferred

- **W1.4 — naklios.dev Immersive mirror.** NakliData's multi-file build
  doesn't match the existing single-file mirror pattern in
  `sync-mirrors.sh`; needs (a) a GitHub Pages deploy workflow here +
  (b) a `source_url` / `pages_url` extension to `apps/manifest.json` +
  `sync-mirrors.sh` in `nakli-dev`. Tracked as its own wave after
  v1.1.0.
- **W1.6 — Map cell basemap.** Stretch; touches privacy posture (third-
  party tile server network calls), warrants a real decision entry.
  Deferred until requested.
- **Custom-endpoint sidecar** (W2.3). Local llamafile / vLLM support
  is blocked on a CSP rework — today's explicit-host whitelist won't
  survive runtime URL configuration.
- **Excel + read_stat DuckDB extensions** — blocked on DuckDB-wasm
  version bump (not published for v1.1.1/wasm_eh).
- **SQLite ATTACH on wasm** — extension vendored, but `ATTACH` fails
  due to a duckdb-wasm VFS bridge limitation. Mount works in native
  DuckDB; defer auto-load until upstream bridges.
