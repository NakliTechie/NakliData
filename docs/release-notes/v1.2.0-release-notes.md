# NakliData v1.2.0 — release notes (draft)

**Tag date:** 2026-05-31 *(draft — not yet tagged)*
**Tagger commit:** TBD (HEAD at tag time)
**Previous tag:** v1.1.0 at `04feedc` (2026-05-24)
**Scope:** 66 commits since v1.1.0.

---

## TL;DR

v1.2 turns the workbench into a product-analytics-shaped tool and a
sharable artifact. **Wave 4** (product analytics surface) adds seven
event-shape semantic types, five pre-built report templates, two new
chart types (funnel + paths), and a new cell kind (cohort) for
reusable user-id filters. **Wave 5** (borrowed-from-the-giants) lands
sensitivity labels on every TypeSpec (Unity Catalog), an assertion
cell kind (dbt-tests), per-column quick-chart suggestions (Power BI
quick measures), and two more sidecar jobs: NL → SQL (Genie / Cortex
/ Magic) and result-summary cards (Hex Magic). **Wave 6** (workflow
polish) adds presentation mode (Hex app-publish), static-HTML export
(Evidence Dev), an interactive input cell (Observable viewof), and a
dashboard layout cell (Superset). Real-Excel mount lands via SheetJS.
GitHub Pages + Cloudflare deploys go live.

Two cell kinds in v1.1 → nine cell kinds in v1.2: SQL, markdown,
chart, pivot, map, cohort, assertion, input, dashboard. Four sidecar
jobs → six. 41 taxonomy types → 48. 13 supported file formats → 15
(GeoJSON / KML in v1.1; xlsx via SheetJS in v1.2).

**Demo verification keystone ran on a real-world xlsx**
(`Retention Rate Analysis_Ecommerce.xlsx`, 14 sheets / 85 columns).
That probe surfaced six latent bugs — all fixed during the same
session, with regression tests added for each.

Privacy posture is unchanged: data never leaves the tab. New network
egress in v1.2: tile.openstreetmap.org for the opt-in map basemap
(images only, no scripts), and the BYOK sidecar for the two new
jobs (only the workbook schema + the user's question / sample
results are shipped, never row data en masse).

## What's new

### Wave 4 — Product analytics surface (W4.1 → W4.5)

NakliData becomes a credible product-analytics tool without touching
the no-server / no-ingestion / no-streaming Hard NOTs.

- **W4.1 — Event-shape taxonomy seeds** (`a32bbbd`). Seven new types
  + a `product-analytics` domain: `event_name`, `user_id`,
  `session_id`, `event_properties_json`, `utm_source`, `utm_medium`,
  `utm_campaign`. `iso_datetime` + `unix_timestamp_ms` headers
  extended with `event_timestamp` / `event_ts`. Total: 41 → 48 types.
- **W4.2 — Pre-built analytics templates** (`98eb283`). Five new
  report templates: DAU, Top events by user-count, Funnel A → B → C,
  30-day retention (cohort), Conversion rate by source. ALL_TEMPLATES
  grew 6 → 11.
- **W4.3 — Funnel chart type** (`ec82668`). Custom SVG renderer —
  horizontal bars with drop-off % annotations. `FUNNEL_A_B_C`
  template rewired to use it.
- **W4.4 — Cohort cell** (`d661c4b`). New `kind: 'cohort'`.
  Structurally a SQL cell whose result is a single `user_id` column;
  downstream cells reference via `@<cohort_name>`. Same execution
  path as SQL — distinct chrome (header label, count badge).
- **W4.5 — Top-K paths chart** (`ec82668`). New TOP_PATHS template +
  the path-chart renderer. Sankey deferred until a real workload
  demands d3-sankey.

### Wave 5 — Borrowed-from-the-giants (W5.1 → W5.5)

Each item maps a Databricks/Snowflake/Microsoft Fabric/Hex ergonomic
onto the workbench. Sequenced smallest-leverage-first:

- **W5.4 — Sensitivity labels in the taxonomy** (`faa3ff3`,
  amendment [A15](./spec-amendments.md#a15--sensitivity-field-on-typespec-w54-amends-spec-32)).
  Each TypeSpec carries a `sensitivity: 'public' | 'pii' | 'financial'
  | 'secret'` field. Schema panel renders a small badge on non-public
  columns. Today: 9 PII, 18 financial, 21 public across 48 types.
  Substrate for future sidecar prompt redaction + demo-mode masking.
- **W5.5 — Assertion cell kind** (`ab3d124`, amendment
  [A16](./spec-amendments.md#a16--four-new-cell-kinds-cohort-assertion-input-dashboard-amends-spec-33)).
  SQL that should return 0 rows when an invariant holds. PASS pill
  on green; FAIL pill + counter-example count + red border on red.
  dbt's `tests:` block analog.
- **W5.3 — Aggregation suggestions in the schema panel** (`c39631a`).
  Power BI quick-measure pattern. Each column row has a "Quick chart
  ▾" affordance that emits ready-to-run SQL + chart + markdown cells
  based on the assigned type and same-table partners. Handles:
  amount / duration_ms / percentage (sum-by-partner + histogram),
  low-card categorical (count-by), date types (count over time),
  user_id / session_id / uuid (COUNT DISTINCT), gstin + amount
  (state-spend via SUBSTR).
- **W5.2 — Sidecar Job 6: Result-summary cards** (`c4fe272`).
  Hex Magic pattern. After a query runs, a "Summarise" button asks
  the sidecar for a one-line observation. Hallucination guard:
  any column reference must be backtick-wrapped AND match a real
  result column — otherwise the entire observation is dropped.
  200-char cap with ellipsis truncation.
- **W5.1 — Sidecar Job 5: NL → SQL** (`df69a73`). Genie / Cortex
  / Magic pattern. "Ask in plain English" toolbar button → modal →
  generates a DuckDB SELECT against your mounted tables.
  **Never auto-executed** (Hard NOT #4). Safety guards in the
  parser (defense in depth alongside the prompt): must start with
  SELECT / WITH, drops anything containing INSERT / UPDATE / DELETE /
  CREATE / DROP / ALTER / TRUNCATE / MERGE / CALL / ATTACH /
  COPY / EXPORT / VACUUM / PRAGMA, drops responses that reference
  tables not in the workbook. Only table + column names ship to the
  sidecar; no row data.

Sidecar job count: 4 → 6. Eval gate: 42/42 → 60/60.

### Wave 6 — Workflow polish (W6.1 → W6.4)

- **W6.2 — Presentation mode** (`64c378b`, amendment
  [A17](./spec-amendments.md#a17--presentation-mode-w62-amends-spec-38)).
  `?present=1` URL param flips into a read-only "deck" view via the
  `app-present-mode` class. Hides sidebars, toolbar, cell-add row,
  per-cell chrome, and SQL/cohort/assertion cells. Markdown + chart
  + pivot + map + dashboard keep rendering. "Exit presentation" pill
  in the header strips the param and reloads. Hex app-publish pattern.
- **W6.3 — Static-HTML export** (`e1ebb42`, amendment
  [A18](./spec-amendments.md#a18--static-html-export-w63-amends-spec-34)).
  New "Export HTML" header button serializes the live notebook DOM
  into a single self-contained `.html` file: markdown previews +
  chart SVGs + pivot/result tables + SQL `<details>` blocks. ~3 KB
  embedded CSS, no JS, no engine. Evidence Dev pattern.
- **W6.1 — Interactive-input cell** (`8914725`). New `kind: 'input'`.
  Text / number / date / select widget. Value inlines into downstream
  SQL via `@<name>` ref resolution — text → quoted (quote-doubling
  for SQL injection safety), number → bare, date → `DATE 'YYYY-MM-DD'`,
  empty → `NULL`. Observable `viewof` + Briefer pattern.
- **W6.4 — Dashboard layout cell** (`1c3bd74`). New `kind: 'dashboard'`.
  CSS grid (1–4 columns) of named markdown / chart / pivot / map
  cells. Items by `@name`; the dashboard re-renders each referenced
  cell with no-op handlers + strips `.cell-head` / `.cell-actions`.
  Superset / Power BI pattern.

Markdown + chart cells gained name inputs as a prerequisite (so
dashboards can reference them).

### Excel mount via SheetJS (`3f20a0b`)

The deferred DuckDB `excel` extension isn't published for our DuckDB-
wasm revision and we shouldn't wait. SheetJS (Apache-2 licensed,
~600 KB lazy chunk) parses `.xlsx` in-browser and emits CSV per
sheet; each sheet flows through the existing `registerCsv` path. Tied
together with the [SheetJS `rawNumbers: true` fix](#sheetjs-rawnumbers-the-silent-xlsx-varchar-everywhere-bug) below.

### Demo verification keystone (`efeafa7`)

New `scripts/verify-demo-ecommerce.mjs` boots `dist/` under headless
Chromium, mounts the queued `Retention Rate Analysis_Ecommerce.xlsx`
via Playwright's filechooser, captures classification + template fit
+ run-all results, and writes evidence to
`plan/demo-verification-ecommerce.json`. Re-runnable against any
file: `node scripts/verify-demo-ecommerce.mjs [path]`. This is the
keystone that converted "Wave 4 ships" from build-time evidence to
real-world-data evidence — and surfaced six fixable bugs along the
way (below).

## Bug fixes (audit + verification surfaced)

The demo-verification keystone surfaced six bugs. Six fixes shipped:

### COLUMN_PROFILE template emitted `your_table` placeholder (`efeafa7`)

The fallback template `requiredTypes: []` made it always applicable.
Its body had a literal `your_table` placeholder that DuckDB rejected
on Run. Removed from `ALL_TEMPLATES`; the templates panel now shows
the helpful empty state ("Mount sources with recognized columns…")
when nothing real applies.

### `notebook.runAll()` ran the seed empty SQL cell (`efeafa7`)

`runAll` looped every SQL cell unconditionally — including the empty
seed cell the notebook ships on first mount. Surfaced a noisy
"Parser Error: syntax error at end of input." Now skips empty cells,
matching Jupyter / Hex / Observable behavior.

### SheetJS rawNumbers — the silent xlsx-VARCHAR-everywhere bug (`2168a14`)

`XLSX.utils.sheet_to_csv` defaults to emitting each cell's FORMATTED
display string. Numbers with thousand separators came through as
`"830,706"`, percentages as `"55%"`, locale dates as `"01/07/25"` —
every column DuckDB's CSV sniffer typed as VARCHAR. Every numeric
column in any real-world xlsx silently went semantic-dead-end. W4
amount/percentage/duration detectors never fired on Excel data.

Fix: `{ rawNumbers: true, dateNF: 'yyyy-mm-dd' }` on `sheet_to_csv`.
After: numbers come through as `830706` / `0.55`; DuckDB infers
BIGINT / DOUBLE; detectors fire. 3 new vitest regression tests.

### Taxonomy detector calibration fallout

- **`user_id` weights** (`fe2af0c`). Header was 0.6 / distribution
  was 0.4 with `high_cardinality: true`. Real events have many rows
  per user; cardinality multiplier of 0.2 pulled confidence to ~0.68,
  just below the 0.7 resolution bar. Rebalanced to 0.8 / 0.2 (no
  high_cardinality requirement). Now 1.0 confidence on real events.
- **`url` regex** (`fe2af0c`). Was `^https?://...$` — `page_url`
  values like `/products` never matched. Extended to also accept
  relative paths.
- **`percentage` detector** (`2b8e738`). Patterns missed
  `Sent_Retention` / `Open Retention` / etc. Extended to include
  retention / ratio / share / conversion. Renamed display from
  "Percentage (0–100)" to "Percentage" (the type already covered
  both fraction and whole-number shapes via range_numeric [0, 100]).
- **`iso_date` / `iso_datetime` regex** (`935ef1e`). Were strict
  ISO 8601. Locale-stored dates (DD/MM/YY, YYYY/MM/DD, dot-separated,
  …) never classified. Regex extended to accept both ISO and common
  locale shapes.

### Audit-surfaced second-order fixes (`fa37310`)

After Wave 5/6 shipped, a forward-pass agent reviewed the new code
and surfaced four bugs:

1. **export-html.ts** — SQL cell with textarea-fallback editor
   exported stale SQL (`.textContent` returns initial content node,
   not user's typed `.value`).
2. **dashboard-cell.ts** — crashed on hand-edited `.naklidata`
   missing `items` or `columns`. Defensive defaults added.
3. **input-cell.ts** — live-typed value invisible to Run-all until
   field blurred (`change` listener only, no `input` listener).
4. **export-html.ts** — "Summarise" button label leaked into the
   exported `<details>` summary line.

All four fixed.

### `@cellName` cycle detection (`5b690c6`)

The `@-ref` rewriter previously sent unresolved references straight
to DuckDB, which threw an opaque "table cell_<id> not found." A
static check before each run now catches three broken shapes:
self-reference (cell named `x` containing `@x`), cycle
(`a → b → … → a`), and unknown reference (`@x` with no matching
cell). All three render clean inline error messages with hints.
20 new vitest cases.

## Privacy posture (unchanged)

Your data never leaves the tab. New egress points in v1.2:

- **tile.openstreetmap.org** when the user explicitly opts into the
  map basemap (image tiles only, no scripts; CSP carve-out for that
  one host).
- **BYOK sidecar provider URL** for the two new jobs (NL → SQL + 
  result summary), only when the user has enabled the sidecar.
  Privacy shape:
  - NL → SQL ships table names + column names; never row data.
  - Result summary ships column names + first 5 sample rows (capped
    by the caller regardless of result size).
- No persistent storage of BYOK keys outside the sessionStorage
  default (opt-in IDB plaintext with a "Forget" affordance —
  unchanged from v1.1; spec amendment A2).

## Breaking changes

**None.** v1.2 is fully backward-compatible:

- The `.naklidata` v1.0 format is unchanged. New optional fields
  (`user_types`, `override_rules`, sensitivity badges in the schema
  panel) round-trip cleanly through v1.1 readers. Older `.naklidata`
  files load without modification.
- The `taxonomy/v0.1` bundle is additive — every v1.1 type id is
  still present (we added 7 product-analytics types, rebalanced
  `user_id` detector weights, extended `url` regex, calibrated
  `percentage` patterns; no removals).
- The sidecar JSON contract is additive — new job kinds
  (`summarise-result`, `nl-to-sql`) join the existing union;
  responses for the old jobs are unchanged.
