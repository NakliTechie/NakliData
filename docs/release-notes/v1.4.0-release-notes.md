## TL;DR

A focused **BI-parity release**. v1.4.0 ships the entire nine-feature
slate (F1–F9) the warehouses/BI/CDP competitive analysis ranked, plus the
infra reconcile that moved deploy to Cloudflare Workers and put the e2e
suite in the CI gate — and one latent v1.3 bug fixed along the way.

- **Semantic layer (F1–F3)** — named **dimensions** (`DIM()`) alongside
  measures, a unified **Semantic** catalog panel, and a JSON **code view**
  of the layer.
- **Authoring ergonomics (F4–F6)** — **calculated / derived fields** on
  any result, **window/LOD** fields, and a **multi-join** visual query
  builder.
- **Profiling (F7–F8)** — a one-click **X-Ray** profile and
  **numeric distribution + outliers** in the column profile.
- **Distribution (F9)** — an **Embed** button that emits a sandboxed,
  server-free `<iframe srcdoc>` of the read-only export.
- **Latent M2 fix** — the lazy measures panel carried its own store
  singletons, so panel-defined measures never reached the notebook;
  broken since v1.3 M2, fixed by inlining the panel.
- **Infra** — GitHub Pages retired (Cloudflare Workers is canonical);
  Playwright e2e added to the CI verify gate.

Feature release. **No breaking changes** — all additive; pre-v1.4
`.naklidata` files round-trip cleanly (the new `dimensions` field is
optional). **No new spec amendments** — every feature extends an
already-amended surface (semantic layer, visual query builder A29,
column profile, export-HTML A18); the design decisions are logged in
`DECISIONS.md` (entries AI–AO).

## Headline — Semantic layer (F1–F3)

### Named dimensions (F1)
Reusable derived dimensions referenceable like measures —
`DIM(gstin_state)` expands to `SUBSTR(gstin, 1, 2)`, `DIM(month)` to a
`date_trunc`. New `src/core/dimensions.ts` (a `DimensionDefinition` store
mirroring measures). `expandMeasures` now expands `DIM(name)` in the
**same pass** as `MEASURE(name)` via an optional `dimensions` param
(back-compatible) with an `unknownDimensions` result field. Pure
client-side macro expansion, single audited expansion point. Dimensions
persist in `.naklidata`. (DECISIONS AI.)

### Semantic catalog panel (F2)
The Measures button became a **Semantic** panel managing **both**
measures and dimensions — add / list / delete, with a "used by N cells"
count. One place to browse and version the whole semantic layer instead
of measures hiding behind a single button.

### Code view (F3)
A "View as code" toggle renders the layer as editable JSON
(`{measures, dimensions}`) with Apply-validation. It is the **store
shape**, not a new DSL — a code-reviewable metrics artifact round-tripped
in the workbook description, never the data (Cube / LookML pattern,
client-side). (DECISIONS AK.)

## Headline — Authoring ergonomics (F4–F6)

### Calculated / derived fields (F4)
A **Calc field** button on every SQL result opens a modal with two modes.
**Expression** mode: a free expression + column chips. The emitted cell
is `SELECT *, (<expr>) AS "<alias>" FROM (<upstream_sql>) AS calc_src` —
the upstream query wrapped as a subquery so the new field composes over
any result. New `src/core/calc-field.ts` reuses the M5 injection-safe
emitter (`quoteIdent` for the alias, `validateMeasureExpression` for the
expression). Output is a new SQL cell — you click Run (Hard NOT #4).
(DECISIONS AL.)

### Window / LOD fields (F5)
The same modal's **Window (LOD)** mode builds Tableau-style
level-of-detail calcs: function + column + optional `PARTITION BY` —
e.g. `SUM(total_amount) OVER (PARTITION BY vendor_name) AS vendor_total`.
Safe-by-construction (function allowlist + quoted identifiers), no free
SQL fragments. (Single-column partition; multi-column is a deferred
stretch.)

### Multi-join visual query builder (F6)
Grew the v1.2 M5 builder (single table + one optional join) toward
Metabase's question builder. The transient spec's `join` became a
`joins[]` array; `buildFrom` loops and `validateSpec` enforces that each
join attaches to an **in-scope** table (the source or an earlier join).
Every filter / aggregate column picker is now **table-qualified** (a
table select spanning in-scope tables + a column select) so joined-table
columns are reachable. Output still goes to a SQL cell — you click Run.
(DECISIONS AM. Multi-step derived pipelines — filter → summarise →
re-summarise — remain a deferred stretch.)

## Headline — Profiling (F7–F8)

### X-Ray profile (F7)
An **X-Ray** button on every SQL result inserts a markdown header + a
**stats cell** bound to the result and runs it — one click gives
descriptives + a correlation matrix (reuses the v1.3 M4 stats cell)
rather than building a new profiling engine. (DECISIONS AN.)

### Numeric distribution + outliers (F8)
`Engine.profileColumn` now adds a five-number summary
(min / q1 / median / q3 / max) + an IQR-rule outlier count, computed over
`TRY_CAST(col AS DOUBLE)` so it's **type-agnostic** — a non-numeric
column yields zero castable values → `numeric: null`; a numeric (or
numeric-string) column gets quartiles. One CTE query + a correlated
outlier subquery, wrapped in try/catch. The schema-panel column profile
renders a **Distribution** + **Outliers** row. (DECISIONS AN.)

## Headline — Distribution / embed (F9)

### Embeddable read-only widget (F9)
An **Embed** button wraps the existing self-contained **Export-HTML** doc
(markdown previews + chart SVGs + result tables, no JS, no engine) in a
**sandboxed `<iframe srcdoc="…" sandbox>`** snippet for read-only,
server-free embedding in a wiki / intranet. The export carries no
scripts, so the sandbox is **empty** (no `allow-scripts`, no
`allow-same-origin`) — maximally locked down. Chosen over a `?lens=`
iframe, which would render empty charts for any local-data notebook (the
common case) and needs a reachable server. The doc is attribute-escaped
so it round-trips; `buildEmbedSnippet` is a pure string transform.
Trade-off: a static snapshot (not live) + a bulky `srcdoc` — acceptable
for the embed use-case. (DECISIONS AO. New `src/core/embed.ts` +
`src/ui/embed-modal.ts`.)

## Latent fix — M2 split store singletons

The measures panel was a lazy chunk, and esbuild's `splitting: false`
made each lazy chunk self-contained — the panel bundled its **own** copy
of the stores, so `getMeasuresStore()` inside the panel was a different
singleton from the notebook's. **Panel-defined measures never reached the
`MEASURE()` expander — broken since v1.3 M2** (no e2e covered the panel
path). Fixed by importing the panel directly into `main` (non-lazy) and
removing `src/lazy/measures-panel.ts`. (DECISIONS AJ.)

## Breaking changes

**None.** All additive. Pre-v1.4 `.naklidata` files round-trip cleanly —

or format-version change (the bundle budget stays 750 KB, A30).
