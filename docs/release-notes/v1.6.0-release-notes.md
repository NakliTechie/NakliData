# NakliData v1.6.0 — release notes

Tag `v1.6.0` at `origin/main`. A big release: the **Polyglot Workbench** opens
with an in-browser **Python cell**, the **Facet view track** lands six new visual
view types, and data-format coverage broadens to **SPSS/Stata/SAS**, offline
**Parquet + Spatial**, **Arrow**, and resilient **SQLite/CSV**. Plus onboarding.
Versioning is git-tag-based; the in-app `BUILD_VERSION` stays `0.1.0`.

Everything is additive and sovereign — no data ever leaves the tab, old
`.naklidata` files round-trip, and every new runtime is vendored same-origin.

## Headline — Python cell (Polyglot Workbench, Fork 2)

Run **pandas** over your data, in the browser, without installing anything or
sending data anywhere. A new **`python` cell** takes an upstream cell's result,
hands you a pandas DataFrame `df`, and re-registers whatever you leave in `df` as
a queryable DuckDB table — so Python is an escape hatch on the SQL spine, not a
separate environment.

- **Sovereign runtime.** Pyodide 0.27.7 + pandas + pyarrow are **vendored
  same-origin** (~33 MB, fetched at install, integrity-pinned) and loaded lazily
  on first run behind a "Downloading Python…" affordance. No CDN reach at runtime;
  runtime *code* is fetched, *data* never leaves the tab.
- **CSP-clean.** Runs under the app's strict `script-src 'self' 'wasm-unsafe-eval'`
  with zero `unsafe-eval` — Python compiles inside the wasm runtime.
- **Fast + bounded.** The DuckDB↔pandas round-trip is Parquet-based; ~500 ms for
  1M rows in the validation spike. Input is capped (2M rows) to keep the tab within
  memory rather than OOM.
- One data plane, many languages — the R cell is next on the same rails.

## Headline — Facet view track (six new view types)

"One data shape, many views," applied to a result set. New visual cell kinds,
each a lazy chunk (deck.gl deduped into one), pure `.naklidata` config:

- **Embedding** — scatter precomputed x/y (or auto-project via in-browser PCA),
  click a point to find its nearest neighbours (no model download, no round-trip).
- **Network** — a force graph from an edge list, laid out by an in-house
  synchronous Fruchterman engine (CSP-clean, no dependency), click a node to
  highlight its neighbourhood.
- **Knowledge-graph + Weighted** — attributed edges: colour edges categorically
  (click-to-filter legend) or scale width by a weight column.
- **Temporal** — a brushable time histogram; drag a window, count rows inside it.
- **Distribution / Categorical** — auto histogram (numeric) or top-value bars
  (categorical); click a bar to select it.

## Added

- **Python cell** over vendored Pyodide (pandas + pyarrow); result re-registers
  as a queryable DuckDB table. `scripts/fetch-pyodide.mjs` vendors the runtime.
- **Facet views:** Embedding (find-similar + PCA), Network (force graph),
  Knowledge-graph / Weighted (attributed edges), Temporal (brushable timeline),
  Distribution / Categorical.
- **SPSS / Stata / SAS mounts** (`.sav/.zsav/.por/.dta/.sas7bdat/.xpt`) via a
  vendored **ReadStat-wasm** reader compiled in-house (no published DuckDB ext
  exists for wasm).
- **Offline Parquet + Spatial** — the `parquet` and `spatial` DuckDB extensions
  are now vendored, so `.parquet` and GeoJSON/KML mount in the offline/shipped
  build, not just online.
- **Arrow / Feather** mounts — a reader re-frames IPC file → stream so `.arrow`
  files load (previously a silent no-op).
- **Onboarding** — an in-app Help modal, a first-run welcome splash, and a
  searchable illustrated field guide (committed generator, screenshots).
- **Guide** — a "Reports from real datasets" section + a click-to-zoom lightbox
  on every screenshot.

## Changed

- **SQLite mounts** now read via **sql.js** → NDJSON → `read_json_auto`
  (DuckDB-wasm's `sqlite_scanner` VFS can't open a browser-registered file). A
  static `.sqlite` genuinely mounts now.
- **CSV** mounting is resilient (whole-file type inference + skip-and-tally of
  malformed rows) and **auto-detects the header** (a headerless file keeps all its
  rows instead of consuming the first as column names).
- **"+ Add source"** is wired to a mount-options modal; `SHOW`/`DESCRIBE`/`PRAGMA`
  run directly instead of erroring.
- The unsupported-file message now names the supported formats.
- deck.gl is deduped into a single shared lazy chunk across all Facet views.

## Fixed

- **PII-leak guard:** the anonymize export now awaits the taxonomy bundle before
  writing.
- **Brave:** File System Access gating handled — folder-mount steers to the
  working "Add file" path; single-file mount prefers the classic file input.
- A dropped-click bug where editing a cell's code re-rendered the notebook and
  detached the Run button mid-click (`Notebook.patchCellSilent`).
- Walkthrough: the schema panel self-heals on restore; the DuckDB logger is
  silenced.

## Removed

- Nothing user-facing. (The dead `read_stat` community-extension path was replaced
  by the vendored ReadStat-wasm reader.)

## Known limitations

- SPSS/Stata **dates** come through as their raw numeric offset (decoding is a
  future refinement).
- **LZ4/ZSTD-compressed Arrow** files aren't readable in-browser (the JS reader
  has no IPC decompression) — re-export uncompressed, or use Parquet.
- An **all-VARCHAR headerless CSV** is inherently ambiguous; DuckDB's sniffer
  decides.
