# NakliData v1.7.0 — release notes

Tag `v1.7.0` at `origin/main`. **The Polyglot Workbench is complete: R joins Python.**
You can now run **R** in the browser, over your own data, with nothing installed and
nothing uploaded — alongside the Python cell from v1.6.0. Versioning is git-tag-based;
the in-app `BUILD_VERSION` stays `0.1.0`.

Additive + sovereign: no data leaves the tab, old `.naklidata` files round-trip, and the
R runtime is vendored same-origin like everything else.

## Headline — R cell (Polyglot Workbench, Fork 2 complete)

A new **`r` cell**, twin to the Python cell: pick an upstream cell's result, get an R
`data.frame` named `df`, transform it in R, and whatever you leave in `df` becomes a
queryable DuckDB table. SQL is still the spine; Python and R are the escape hatches.

- **Sovereign runtime.** WebR (R compiled to WebAssembly) + the R base library are
  **vendored same-origin** (~66 MB, fetched at install) and loaded lazily on first run
  behind a "Downloading R…" affordance. No CDN reach at runtime; your data never leaves
  the tab.
- **Base R, no package installs.** The DuckDB↔R interchange is CSV over WebR's virtual
  filesystem via base `read.csv` / `write.csv` — so a cell works with zero package
  downloads and stays fully offline-capable.
- **Row-capped** (2M) to keep the tab within memory rather than OOM.

## Under the hood — cross-origin isolation

R needs `SharedArrayBuffer`, so this release turns on **cross-origin isolation**
(`COOP: same-origin` + `COEP: credentialless`). The `credentialless` mode was the key: it
enables `SharedArrayBuffer` **without** breaking the app's cross-origin loads (the DuckDB
engine is cross-fetched from a mirror on the hosted build). Verified — the whole app works
identically under isolation.

This also **unblocks GPU-accelerated Facet graph layouts** (`@antv/layout-wasm`) for very
large networks, a capability that was previously gated on the same requirement.

## Added

- **R cell** over vendored WebR; result re-registers as a queryable DuckDB table.
  `scripts/fetch-webr.mjs` vendors the runtime from the pinned `@r-wasm/webr` package.
- A shared **language-cell** surface now backs both the Python and R cells.

## Changed

- **Cross-origin isolation** (`COOP`/`COEP: credentialless`) is now set on every served
  response (Cloudflare `_headers`, the dev server, and the smoke server), enabling
  `SharedArrayBuffer`.
- The shell bundle budget rose from 750 KB to **768 KB** (spec amendment A35) to cover
  wiring a second language into the shared shell; heavy compute still ships as lazy
  chunks.

## Fixed

- A latent dropped-click bug where editing a language cell's code and immediately
  clicking Run could lose the click (the cell now persists code without a re-render).

## Known limitations

- **R types via CSV:** columns round-trip through CSV, so types are re-inferred both ways
  (e.g. dates come back as strings). A typed interchange is a future refinement.
- **R package installs** would reach the cross-origin R package repository — a sovereignty
  tradeoff, so v1.7.0 ships base R only.
- The hosted build now serves ~110 MB of vendored language runtimes (Python + R); they're
  cached after first load.
