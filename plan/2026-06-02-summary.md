## Decisions

- **`frame-ancestors` shipped in `<meta>` CSP despite being a no-op
  there.** Documented in the inline esbuild config comment. Ships
  for any future header-capable deploy; GitHub Pages can't enforce
  it but the directive is the right shape.
- **Lens-link auto-mount → confirmation dialog** (vs reconnect-tiles
  / vs same-origin-only). User chose confirmation. Cancel-focused so
  Enter-dismiss is the safe default.
- **`xlsx` pinned to exact `0.18.5`** rather than fully exact-pinning
  every dep. Only the unmaintained-on-npm one. `package-lock.json`
  pins the rest.
- **Two-track adversarial review as standing gate.** Internal
  code-reviewer + external `codex review` catch complementary
  blind-spot sets — proven 3 times now (after v1.2.0, after Wave
  5/6, after v1.2.1). Standing post-batch ritual going forward.
- **`mountIcebergTable` keeps the laxer `^https?://` regex on
  user-typed URLs** (intentional local-testing allowance per the
  inline comment). `mountIcebergCatalog`'s catalog-returned URL is
  strictly `^https://` because the user never sees it.
- **Postinstall scripts now `exit(1)` on real failures** (was
  `exit(0)` — silently passing partial/tampered installs).

## Tried then rolled back

- **CACHE_VERSION rewrite using non-global regex.** First-pass
  M12 implementation. Internal code-review caught the single-shot
  miss: a second `CACHE_VERSION = '…'` occurrence in `sw.js`
  (debug log, comment) would have silently left a stale value.
  Fixed with `/g` + assert match-count === 1.
- **sinks.ts SQL-escape on the FILENAME variable.** First-pass L7
  fix doubled `'` in both the SQL literal AND the JS filename
  string. Codex caught it — DuckDB writes the file under the
  unescaped name; `readDuckDbFile` would have looked for the
  doubled name and failed. Fixed by maintaining two distinct
  variables (`rawName` for filesystem, `sqlName` for SQL literal).
- **Iceberg catalog scheme regex `^https?://|^s3://`.** First-pass
  H8 fix. The `?` permits `http://` AND `https://`, contradicting
  the error message and the actual intent. Codex caught. Fixed to
  strict `^https://`.
