## Decisions

- **Promise-chain mutex for `applyLoadedFile`** over Web Locks API —
  12 lines, no dep, single-call-site need. Errors don't couple across
  chained invocations. (DECISIONS 13:00)
- **`.naklidata` format-version policy**: additive optional fields
  don't bump (`user_types` + `override_rules` shipped without one);
  bumps are reserved for breaking required-field shape changes.
  (DECISIONS 14:00)
- **NakliData is an independent product, not a portfolio app.** W1.4
  mirror dropped; cross-repo `sync-mirrors.sh` extension off the
  roadmap. CLAUDE.md "Portfolio rules" section left intact (the
  engineering standards still hold; the discoverability story
  doesn't).
- **CSP `connect-src` broadens from explicit-host whitelist to
  `'self' https:`** (DECISIONS 15:00) — required to make Wave 2's
  user-configured URLs possible at all. Trade-off acknowledged: a
  hypothetical XSS that bypassed the SHA-pinned script-src could
  exfiltrate to any HTTPS host. Script-src stays the actual primary
  defence.
- **`SET s3_*` over `CREATE SECRET`** for S3 endpoints (DECISIONS
  15:30). Wasm 1.1.1 doesn't ship `CREATE SECRET` in a useful form;
  connection-wide config means one set of S3 credentials per session
  (documented limitation, future enhancement).
- **Two SourceKinds for Iceberg**: `iceberg-table` (URL-based, slice
  3a) and `iceberg-catalog` (REST-based, slice 3b). Different mount
  flows, different persistence shapes, cleaner than overloading
  one kind.
- **Iceberg slice 3a + 3b ship Bearer-only.** OAuth2 device flow +
  AWS SigV4 (for Glue) defer to v1.3 alongside enterprise multi-
  tenant work — the OAuth UX is its own modal flow.
- **Doc-cadence pinned in CLAUDE.md**: `YYYY-MM-DD-summary.md` is
  canonical going forward; existing `checkpoint-*-eod.md` files stay
  as historical record.
- **Smart Iceberg table-name derivation** handles the canonical
  `<table>/metadata/v<N>.metadata.json` layout by walking up two
  directories when the parent is literally `metadata`.

## Tried then rolled back

- **First save-load e2e fix: `waitForClassificationStable` only.**
  Captured `before` reliably; failure shifted from "extra columns
  in `after`" to "sources doubled" — exposed the auto-restore vs
  explicit-Load race that became the morning's mutex fix.
- **Second save-load fix: `indexedDB.deleteDatabase('naklidata')`
  between save and reload.** Papered over the race for the tag.
  Reverted post-tag once the production mutex landed; the test
  now exercises the race directly as a regression guard.
- **"Have the load handler await any pending restore" as the
  production fix.** Covered boot-restore-vs-Load but not Load-vs-
  Load (two quick clicks), Load-vs-lens-decode, or session-switch-
  vs-Load. A general mutex over the function covered every pairing.
- **Iceberg table-name derivation, first attempt: parent-directory
  only.** A URL like `…/sales/metadata/v3.metadata.json` gave
  `metadata` as the table name — exposed the canonical layout
  needs grandparent traversal when the parent dir is literally
  `metadata`.
