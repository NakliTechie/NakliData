## Decisions

- **Promise-chain mutex over Web Locks API.** Twelve lines, no new
  dependency, single-call-site need. Pattern: snapshot tail → build
  new promise that `await prev` (swallowing prior rejection — calls
  are independent work) → publish as new tail. (DECISIONS 13:00,
  five sub-decisions.)
- **Errors from a prior `applyLoadedFile` invocation do not block the
  next.** Chain guarantees ordering, not error coupling. Each caller
  gets its own rejection.
- **Save-load e2e reverts the IDB-clear workaround.** Auto-restore
  now fires concurrently with the explicit Load in every run; the
  existing `expect(after.sources).toEqual(before.sources)` is the
  contract. Reverting the mutex makes the test fail with 4 cards
  instead of 2 — that's the regression guard.
- **Tag at HEAD, not at v1.0 + cherry-pick.** v1.1.0 = the
  accumulated arc from `5b10b93..04feedc` (27 commits), not a
  curated subset. The release notes document what's in scope.
- **Release notes' "Internal / build" left untouched after the
  post-tag mutex fix.** Tag state is what it was at tag time; the
  follow-up is documented in STATUS + DECISIONS + pending.md, not
  retconned into the release notes.
- **plan/ stays tracked, not gitignored.** Project convention is
  that `plan/spec-amendments.md`, `plan/pending.md`,
  `plan/v1.1.0-release-notes.md`, `plan/checkpoint-*-eod.md` are
  canonical project docs referenced from CLAUDE.md. The portfolio
  windup default (gitignore plan/) is overridden here.

## Tried then rolled back

- **First save-load fix: `waitForClassificationStable` only.**
  Captured `before` reliably; failure shifted from "extra columns in
  `after`" to "sources doubled in `after`" — exposed the deeper
  auto-restore vs explicit-Load race.
- **Second save-load fix: `indexedDB.deleteDatabase('naklidata')`
  between save and reload.** Papered over the race for the tag (test
  green, gate passed, tag shipped) — but the production bug was
  real. Reverted post-tag once the mutex landed; the test now
  exercises the race directly.
- **"Have the load handler await any pending restore" as the
  production fix.** Covered the boot-restore-vs-Load case but not
  Load-vs-Load (two quick clicks), Load-vs-lens-decode, or
  session-switch-vs-Load. A general mutex over the function covers
  every pairing without per-caller bookkeeping.
