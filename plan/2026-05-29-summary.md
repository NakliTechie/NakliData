## Decisions

- **Eval harness: esbuild-bundle the TS runner, not add `tsx`.** Node
  22's native type-stripping rejects TS parameter properties
  (`SidecarError`); esbuild (already a devDep) bundles `harness.ts` +
  its `src/` imports. Reuses the REAL prompt builders + parsers; calls
  the provider transport directly with an env key (dispatchJob's
  loadKey is browser-only). (DECISIONS 21:30.)
- **Eval fixtures use all-passing recorded responses;** scorer pass/fail
  discrimination is unit-tested separately — keeps `--dry-run` a clean
  self-test (exit 0) instead of always-failing.
- **Job 4 ranks only already-applicable templates** (deterministic
  type-gating decides what's *possible*; the sidecar only reorders
  what's *promising*). Hallucination guard in the PARSER, not just the
  prompt. Opt-in, never auto. Ephemeral ranking. (DECISIONS 22:00.)
- **Local model: privacy-first, NO silent BYOK fallback.** pending.md
  said "fallback to BYOK when not downloaded," but silently shipping
  the user's schema to a cloud provider when they picked `'local'`
  violates that choice. An unloaded local model throws an actionable
  error; the one-click provider switch is the explicit fallback.
  (Divergence logged in A11 + DECISIONS 22:30.)
- **Local model library: `@huggingface/transformers`** (maintained
  successor to `@xenova/transformers`); lazy chunk only, never in the
  600 KB shell. Dep NOT added yet — lands with slice B.
- **Seam-now / model-next slicing** for W3.2: the verifiable seam ships
  now; the real model needs a real browser + WebGPU (not
  headless-verifiable), so it's a dedicated follow-up session.
- **Compute Bridge browser client uses HTTP + Arrow IPC, not Arrow
  Flight.** Browsers can't do native gRPC. Flight stays canonical for
  non-browser clients. Bridge query results land via NakliData's
  existing `registerArrow` path → become local DuckDB tables. (W3.3
  design.)
- **Settings `'local'` radio gated to slice B.** Persistence accepts
  `'local'` now (round-trips), but exposing a toggle that always errors
  "not loaded" is poor UX — the radio appears when the model works.
