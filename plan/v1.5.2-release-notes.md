# NakliData v1.5.2 — release notes

Tag `v1.5.2` at `origin/main`. The third and final surface of the **Resolve
track** — **M3: the golden-table sink** (the *Own* verb). With it the track is
complete end to end: **resolve → segment → own.** Additive minor; versioning is
git-tag-based (`BUILD_VERSION` stays `0.1.0`).

## Headline — Export golden table

A new export **sink** (alongside CSV / Parquet / Export-anonymized / KanZen /
Bahi / NakliPoster) that collapses a result to **one row per canonical entity**
and writes the deduped table to a folder you keep, as CSV or Parquet:

    SELECT "vendor__merged",
      first("name")            AS "name",
      max("amount")            AS "amount",
      arg_max("status", "ts")  AS "status"
    FROM <result>
    GROUP BY "vendor__merged"

- **Survivorship per column.** Pick the entity (group-by) column — typically a
  clustered `__merged` column from *Cluster values* (M1) — then a rule per other
  column: **keep-first / max / min / latest**. `latest` keeps the row with the
  MAX of a chosen order column (`arg_max`).
- **Injection-safe by construction.** Every identifier flows through
  `quoteIdent`; the aggregate function comes from a fixed allowlist, never
  templated from user input. The entity column is the GROUP BY key, excluded
  from the aggregate list.
- **A file you keep.** Customer 360, inverted to ownership — written only to the
  folder you pick via the file picker; nothing leaves the tab except into your
  own disk. No profile-as-a-service, no plane to push to.
- Configured in a modal with an entity picker, per-column survivorship selects,
  an order-column picker (shown when any rule is `latest`), a CSV/Parquet
  toggle, and a live SQL preview. The sink mirrors the Export-anonymized write
  path (`COPY (<sql>) TO <temp>` → read → folder-write).

## The Resolve track, complete

- **M1 — Clustering / fuzzy-merge** (v1.5.0): resolve variant spellings into a
  canonical `__merged` column.
- **M2 — Segment primitive** (v1.5.1): named, reusable `SEGMENT(name)` audiences.
- **M3 — Golden-table sink** (v1.5.2): own the resolved canonical-entity table as
  a file.

NakliData's sovereign, file-owned answer to an agentic CDP's resolve → segment →
own loop — done without anything leaving the tab.
