# NakliData v1.4.1 — release notes

Tag `v1.4.1` at `973d416` (on `origin/main`). A patch release closing the
batch of work that landed after the v1.4.0 tag but before the Resolve track.
Versioning is git-tag-based; the in-app `BUILD_VERSION` stays `0.1.0` (a static
build stamp, as it has since v1.0).

**Why a patch (not a minor):** the batch is bug-fix-dominant. The two feature
touches (F5, F6) are stretches on already-shipped surfaces (F4 calc fields, F6's
own visual builder), not new top-level capabilities — so this ships as v1.4.1,
reserving v1.5.0 for the Resolve track's first new surface (M1). See DECISIONS
BA.

## What's in it (since v1.4.0)

- **F5 — multi-column window partitions.** The calc-field window (LOD) builder
  now partitions by multiple columns (checkbox group); the core was already
  array-capable.
- **F6 — multi-step query pipelines.** The visual query builder grew derived
  steps (filter → summarise → re-summarise) that nest the base as
  `FROM (…) AS step_N`, reusing the injection-safe emitter.
- **Parked-minor fixes** (forward-pass follow-throughs): M14 (runAll now runs in
  `@name`-dependency / topological order), M17 (aggregate-without-GROUP-BY
  column drop), M19 (lineage orphan-edge on cell delete), M9/M10 (measures-panel
  Enter-to-add; `window.confirm` → reusable confirm modal), plus the L/S batch.
- **Local-model sidecar — load + registration fixed.** WebGPU device path +
  `q4f16` so the 1–2B models load; the split-singleton registration bug fixed so
  loaded local jobs actually dispatch. Local provider remains **experimental**
  (in-browser inference quality is a tracked v1.5.x R&D item); the cloud BYOK
  provider is the default and works.
- **Cloud-BYOK smoke coverage** for the sidecar path (mocked transport).

No breaking changes; no `.naklidata` format change. Gates at the tag: 819 vitest
