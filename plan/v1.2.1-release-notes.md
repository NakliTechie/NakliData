## Bug fixes

### Dashboard-embedded cells leaked as duplicates in exports (`d6befcb`)

`buildStandaloneHtml` walked `notebookRoot.querySelectorAll('.cell')`,
which matched EVERY descendant `.cell` element — including the
embedded copies re-rendered inside `.dashboard-slot` (the dashboard
re-invokes the per-kind renderer with no-op handlers; those copies
still carry the `.cell` class).

Net effect: a notebook with a dashboard containing a markdown +
chart cell exported BOTH at the top level AND duplicated them
inside the dashboard slot, AND silently dropped the dashboard grid
itself because the exporter had no `kind === 'dashboard'` branch.

**Fix.** Walk children of `.notebook` directly (top-level only) +
add a `dashboard` branch that emits the grid with inner svg /
markdown / table content inline.

### Markdown cell in edit mode exported as empty section (`d6befcb`)

The exporter read `.markdown-preview` for markdown content, which
only exists when the cell is in preview mode. A cell currently being
edited shows a `<textarea>` instead — the exporter emitted an empty
`<section>` for it.

**Fix.** Fall back to `textarea.value` when `.markdown-preview` is
absent, then re-run the same markdown renderer the cell uses.
Exported `renderMarkdownToHtml` from `src/ui/cells/markdown-cell.ts`
(was module-private).

### Bonus tightenings while in the area

- **Input cells now serialise as a labelled chip** in the export
  (e.g., "**Vendor name** Acme") so the static doc shows the
  parameter the rest of the notebook was computed against.
  Previously silently dropped.
- **Empty dashboard slots preserve the affordance text** in the
  exported HTML ("No cell named X" / "only markdown / chart / pivot /
  map can be embedded") so the reader sees the same hints they'd see
  in the live workbench.

## Why an external code review found bugs the internal one didn't

The internal forward-pass agent that audited the v1.2.0 surface
caught five bugs (all fixed in v1.2.0). After tagging, we ran
`codex review --base v1.1.0` as a second-opinion adversarial pass.
It surfaced two more.

The pattern is worth keeping: different tools, different blind spots.
The internal agent's report focused on cell-state mutations and
defensive-default oversights; the external review noticed cross-
component interactions (dashboard's embedded `.cell` class leaking
into export-html's DOM walker) that the internal agent didn't.
