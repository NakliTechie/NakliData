## Decisions

- **Bundle cap raised from 600 KB → 750 KB** at v1.3 M1 (`a0fa5cf`).
  Rationale (autonomous call): v1.3 adds 6 new surfaces; the original
  600 KB came from v1.0 when scope was much smaller; lazy-load remains
  the default for new heavy panels; raised cap covers the shared
  shell's accumulated surface, not a license to dump deps. **Owed:**
  the spec amendment A30 that documents this — flagged as Critical
  C2 in the audit. Batch A keystone.
- **v1.3 M1, M3, M5, M6 shipped as "Phase 1 pure logic + tests."**
  Each gate's load-bearing test invariant is captured (taxonomy
  default matrix for M5; round-trip invariant for M6 — though see
  C3 below; airtight predicate builder for M1; print CSS for M3).
  UI extensions deferred to Phase 2 (manual associations panel,
  cell-ref embedding, shelf drop-zones, lineage edit-mode toggle).
  Bundle headroom (142 KB on the raised cap) makes Phase 2 ships
  comfortable when scheduled.
- **Engine-boundary contract = pure logic only, no DOM/FSA/browser
  globals.** Refactored `taxonomy/load.ts` to inject the fetcher;
  moved `anonymize.ts` from `src/ui/sinks/` to `src/core/`; extracted
  ChartConfig schema to its own module (`src/core/chart-config.ts`)
  so the M5 Transparency Rule "one schema, three producers"
  invariant has a single source of truth.
- **M5 measure expression contract = filtered aggregate SQL fragment
  in SELECT-list position.** Uses DuckDB's `FILTER (WHERE ...)` clause
  so a measure body composes cleanly into a parent SELECT. Example:
  `revenue = SUM(amount) FILTER (WHERE status = 'completed')`.
  Single audited macro-expansion point in `src/core/measures.ts`.
- **M3 PDF export = browser print-to-PDF, NOT pdf-lib.** Print CSS
  scopes visibility per the report cell at `@media print`. Zero new
  runtime dependencies. (Audit found H10 print-scope leak + H11
  cell-ref-embedding-TODO — Phase 2 wire.)
